## 🔐 Система Аутентификации

### Концепция
```
1. POST /admin/login → получить токен
2. Использовать токен в заголовке Authorization: Bearer <token>
3. Мидлвэр requireAdmin проверяет токен перед каждым запросом
```

### Учетные данные
- **Переменная окружения**: `ADMIN_PASSWORD` (по умолчанию "admin2026")
- **Формат токена**: `admin-token-{пароль}`
- **Запрос токена**: POST /api/admin/login с JSON телом `{"password":"..."}`

### Пример Потока
```bash
# 1. Вход
$ curl -X POST http://localhost:3000/api/admin/login \
  -H 'Content-Type: application/json' \
  -d '{"password":"admin2026"}'

# Ответ:
# {"token":"admin-token-admin2026","message":"Login successful"}

# 2. Использование токена для защищённого endpoint
$ curl -H "Authorization: Bearer admin-token-admin2026" \
  http://localhost:3000/api/admin/stats

# 3. Без токена → 401 Unauthorized
$ curl http://localhost:3000/api/admin/stats
# {"message":"Unauthorized"}
```

### 2️⃣ POST /admin/bots
**Статус**: ✅ Работает | Требует авторизацию

Создание нового бота в каталоге.

```bash
curl -X POST http://localhost:3000/api/admin/bots \
  -H 'Content-Type: application/json' \
  -H 'Authorization: Bearer admin-token-admin2026' \
  -d '{
    "username": "@mybot",
    "name": "My Bot",
    "description": "Cool bot",
    "categoryId": 1,
    "rating": 4.5,
    "reviewCount": 10,
    "isVerified": true,
    "isPremium": false,
    "tags": ["utility", "fun"],
    "monthlyUsers": 1000,
    "iconEmoji": "🤖",
    "telegramUrl": "https://t.me/mybot"
  }'
```

**Побочные эффекты**:
- ✅ Автоматически увеличивает счётчик категории (+1)
- ✅ Сохраняет категорию в отдельном поле для быстрого доступа

| Статус | Значение |
|--------|----------|
| 201 | ✅ Бот создан |
| 400 | ❌ Ошибка валидации или категория не найдена |
| 401 | ❌ Не авторизован |

## 🐛 Выявленные Проблемы

### 🔴 Высокий Приоритет

#### 1. **N+1 SQL Запросы**
**Файл**: admin.ts:195-265

Описание: Для каждого бота выполняется несколько отдельных SQL запросов.

**Пример**:
- 100 ботов = 600+ SQL запросов
- Очень медленно при масштабировании

**Рекомендация**: Использовать агрегацию на уровне SQL.

#### 2. **Нет Rate Limiting на Login**
**Файл**: admin.ts:33

Описание: Нет защиты от брутфорса пароля.

**Рекомендация**: Использовать `express-rate-limit` для ограничения количества попыток.

### 🟡 Средний Приоритет

#### 3. **Небезопасное Хранение Пароля**
**Файл**: admin.ts:16-17

```typescript
const ADMIN_PASSWORD = process.env.ADMIN_PASSWORD || "admin2026";
```

Проблема: Пароль по умолчанию жёстко закодирован.

Рекомендация: Требовать обязательное наличие переменной, выбросить ошибку если не установлена.

#### 4. **Отсутствие Аудита Логов**
**Файл**: admin.ts (весь файл)

Описание: Нет логирования кто и когда изменил ботов.

Рекомендация: Логировать все операции CRUD с IP администратора и временем.

#### 5. **Токен Содержит Пароль**
**Файл**: admin.ts:17

```typescript
const ADMIN_TOKEN = `admin-token-${ADMIN_PASSWORD}`;
```

Проблема: Токен содержит гранулированный пароль, не хешируется.

Рекомендация: Использовать JWT или другую криптографическую систему.

## 🔧 Как Запустить Сервер

### Требования
- Node.js 24+
- pnpm
- PostgreSQL (DATABASE_URL)

### Шаги

```bash
# 1. Установить зависимости
pnpm install

# 2. Скомпилировать API сервер (уже сделано)
pnpm run build --filter @workspace/api-server

# 3. Запустить сервер

PORT=3000 \
node --enable-source-maps ./artifacts/api-server/dist/index.mjs

# 4. В другом терминале - запустить тесты
node test-admin-panel.js
```
