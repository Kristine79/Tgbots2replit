## 📋 Обзор Админ Панели

Админ панель находится в `/artifacts/api-server/src/routes/admin.ts` и предоставляет следующие возможности:

### Основные Маршруты

1. **POST /api/admin/login** - Вход в админ панель
2. **POST /api/admin/bots** - Создание бота (требует авторизацию)
3. **PUT /api/admin/bots/:id** - Обновление бота (требует авторизацию)
4. **DELETE /api/admin/bots/:id** - Удаление бота (требует авторизацию)
5. **GET /api/admin/stats** - Получение статистики (требует авторизацию)

---

## 🔐 Система Аутентификации

### Учетные Данные
- **Password**: Переменная окружения `ADMIN_PASSWORD`, по умолчанию `"admin2026"`
- **Token Format**: `admin-token-{ADMIN_PASSWORD}`

### Механизм Авторизации
```
- Login отправляет пароль в теле запроса
- API возвращает токен в формате Bearer token
- Токен должен быть отправлен в заголовке: Authorization: Bearer <token>
- Все защищённые endpoints проверяют токен через middleware requireAdmin
```

### Пример:
```bash
# 1. Вход
curl -X POST http://localhost:3000/api/admin/login \
  -H "Content-Type: application/json" \
  -d '{"password":"admin2026"}'

# Ответ:
# {"token":"admin-token-admin2026","message":"Login successful"}

# 2. Использование токена
curl -X GET http://localhost:3000/api/admin/stats \
  -H "Authorization: Bearer admin-token-admin2026"
```

## 🐛 Найденные Проблемы

### Высокий Приоритет

#### 1. **N+1 SQL Запросы в GET /admin/stats**
**Статус**: ⚠️ Производительность

Код:
```typescript
// admin.ts:202-222
await Promise.all(
  bots.map(async (bot) => {
    // Для каждого бота выполняется 4+ отдельных запроса
    const [totalRow] = await db.select()...  // SELECT 1
    const [uniqueRow] = await db.select()... // SELECT 2
    const [last7Row] = await db.select()...  // SELECT 3
    const [last30Row] = await db.select()... // SELECT 4
    const dailyRows = await db.select()...   // SELECT 5
    const geoRows = await db.select()...     // SELECT 6
  })
)
```

**Проблема**: 
- Если 100 ботов → 600+ SQL запросов
- Очень медленно масштаб на больших данных

**Рекомендация**:
- Использовать агрегирование на уровне SQL (GROUP BY, window functions)
- Или кешировать результаты статистики

---

### Средний Приоритет

#### 2. **Небезопасное Хранение Пароля в Коде**
**Статус**: ⚠️ Безопасность

Код:
```typescript
const ADMIN_PASSWORD = process.env.ADMIN_PASSWORD || "admin2026";
const ADMIN_TOKEN = `admin-token-${ADMIN_PASSWORD}`;
```

**Проблема**:
- Пароль по умолчанию `"admin2026"` жёстко закодирован
- Если забыть установить переменную окружения, используется слабый пароль
- Токен содержит сам пароль (не хешируется)

**Рекомендация**:
- Требовать обязательное наличие `ADMIN_PASSWORD`
- Использовать хеширование для токенов (например, JWT)
- Выбросить ошибку с понятным сообщением если переменная не установлена

---
